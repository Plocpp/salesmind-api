Visão Geral do Sistema Salesmind API

Esta página mostra, de forma simples e visual, como o sistema se comporta: quem chama, quais rotas existem, como a autenticação funciona e para onde vão os dados.

Como o sistema funciona

1
O cliente envia uma requisição para as rotas definidas em src/routes.
2
O middleware de autenticação valida o token e marca o pedido com userId e role.
3
Os controllers recebem o pedido e chamam os serviços para executar a lógica de negócio.
4
Os serviços usam Prisma para ler/gravar dados em Usuario e Produto no banco.

O que o sistema faz

  • Autentica usuários com login e refresh token.
  • Registra novos usuários e permite logout.
  • Gerencia produtos: criar, listar, buscar, atualizar e deletar.
  • Mostra um painel administrativo se o usuário for admin.

Principais pontos para qualquer pessoa entender

Rota de autenticação: /auth cuida de login, registro e perfil.

Rota de produtos: /produtos cuida do cadastro e controle de produtos.

Segurança: o middleware garante que apenas usuários autenticados usem as APIs.

Admin: o painel de dashboard só está disponível para quem tem role = "admin".
